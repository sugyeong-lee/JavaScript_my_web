Detached HEAD

  1. Detached HEAD란?

    HEAD가 특정 branch가 아닌 특정 commit을 직접 참조하고 있는 상태

  2. Detached HEAD를 사용하는 이유는?

    보통 해당 repo에서 기록을 남기지 않고 시뮬레이션을 해보고 싶을 때 detached 상태를 이용 But!!! Git에서 지양하는 상태이므로 test branch를 만들어 사용하는 것이 좋다.

  3. Detached HEAD에서 벗어나려면?

    이전 commit버전으로 되돌리거나 임시로 새로운 branch를 만들고 지운다.
    관련 코드
    git branch -f master temp
    git checkout master

  4. 회고

    상황 1. 이번 주는 강의를 탐구하는 것 보다는 git을 사용하는데 더 시간을 쏟은 것 같다. 평화롭게 코딩을 하고 있는 어느 날 갑자기 git이 이상해졌다. “branch를 만들어 github pages를 할 수 있을까?”라는 의문점에서 시작하여 branch를 만들어 있는 와중에 origin/main이라는 것이 떠서 이 명령어?를 적으면 HEAD가 메인을 가리키는 것이라는 착각을 해서 입력을 했다. 그러나 뭔가 평소에 볼 수 없었던 새로운 화면이 나타나서 당황하고 상황을 해결하기 위해 새로운 브랜치를 만들고 등을 계속 시도했다. 상황 2. 알고보니 저번 주 스터디 때 공부했던 진도가 다음 주에 할 스터디 진도였던 것이다………정말 바보 같은 상황이다. 그래서 다른 분이 질문했던 것이 어디에서 나온 질문인지 이해하지 못 했던 것이다. 정말 바보인 것 같다….. 그래서 다시 원래 저번 주에 했어야 했던 진도를 다시 공부했다. 그러니 비로소 저번 주에 이해가 되지 않았던 것들이 이해되었다. 약간의 이상함을 느꼈어야 했는데 눈치 못 챘던 과거의 나는 반성한다. 그러면서 배운 것들을 활용하면서 반성 웹사이트를 만들었다. CSS를 통해 꾸미지 않아서 너무 허전해서 다음 주에 찾아가며 꾸미기로 했다. 그리고 저번 주 원래 진도를 공부하면서 채팅, 응답, 경로 등의 기능을 추가할 수 있는데 아직 오류를 해결하지 못해서 다음 주까지는 CSS와 다른 기능 추가하기까지 꼭 하려고 한다.
